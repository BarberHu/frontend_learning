# Roadmap Overview

Use this file as the strategic snapshot of Wang Yang's frontend job-search learning plan. It keeps the AI aligned before reading the full `docs/learning-map.md`.

## Positioning

Target identity:

```text
React/Next/TypeScript 前端工程师
  + Node 能做全栈闭环
  + Python/GIS 能做数据处理
  + Agent 能做智能化交互与自动分析
```

Do not turn the route into generic CS exploration. The first 6 months should create employable frontend artifacts.

## 6-Month Spine

| Month | Main Focus | Required Artifact |
| --- | --- | --- |
| 1 | JS/TS 基础修复 | 30 个 JS/TS 小练习，能解释闭包、Promise、事件循环、泛型、类型收窄 |
| 2 | React 主线 | React 数据看板，覆盖组件拆分、状态、请求、错误状态、图表 |
| 3 | Next + Node 全栈 | Next.js 全栈 CRUD，打通页面、API、数据库、部署 |
| 4-5 | WebGIS 主项目 | WebGIS 空间数据分析平台一期，覆盖 GeoJSON、图层、筛选、空间统计、图表联动 |
| 6 | Agent + 求职化 | 自然语言空间查询、自动分析摘要、报告草稿、简历、README、Demo、面试稿 |

## Technology Defaults

- Main frontend stack: React, Next.js, TypeScript.
- Secondary framework: Vue only for reading and interview familiarity, not as a parallel main line.
- Backend: Node for full-stack closure.
- GIS/data: Python for preprocessing, Turf.js for browser-side geospatial analysis.
- Web map: Mapbox GL JS or Leaflet; pick one for the first project version.
- AI/Agent: Use Agent features only when they serve GIS tasks, not as a decorative chat box.

## Weekly Operating Rule

Each week should contain only:

```text
1 个主知识点
1 个项目功能
1 组算法题
```

Every week must leave at least one visible artifact: code, README, note, screenshot, review, or commit.

## Decision Guardrails

- If the user asks whether to change routes, judge by ROI for frontend employment and the WebGIS + Agent project.
- Prefer depth in React/Next over parallel React/Vue mastery in the first 6 months.
- Prefer working code and reviewable artifacts over course collection.
- Prefer one deployable project over many unfinished demos.
- Algorithm practice matters, but must not consume the WebGIS project.

## Success Criteria

After 6 months, the user should have:

- One main project: WebGIS + Agent spatial data analysis platform.
- Two support projects: React data dashboard and Next full-stack CRUD.
- A weekly learning log with visible artifacts.
- A resume-ready project narrative.
- Interview explanations for JS/TS, React, HTTP, Node API, map rendering, spatial analysis, and AI output boundaries.
